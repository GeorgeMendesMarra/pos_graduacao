# PMO: Classificação por Abrangência

O Project Management Office (PMO) pode ser classificado de diferentes formas, sendo a **classificação por abrangência** uma das mais importantes para definir seu escopo de atuação, autoridade e relacionamento com a estrutura organizacional. Esta classificação determina quais projetos, programas ou portfólios estão sob a responsabilidade do PMO e qual seu nível de influência na organização.

---

## 📊 Os Principais Tipos de PMO por Abrangência

| Tipo | Abrangência | Autoridade | Reporte | Escopo |
| :--- | :--- | :--- | :--- | :--- |
| **Enterprise PMO (EPMO)** | Organizacional total | Alta | CEO, Diretoria Executiva | Todos os projetos, programas e portfólio |
| **Divisional PMO** | Unidade de negócio/departamento | Média | Vice-presidência da divisão | Projetos de uma área específica |
| **Project PMO** | Projeto ou programa específico | Variável (alta no escopo) | Patrocinador ou comitê do projeto | Projeto ou programa único |
| **Portfolio Management Office (PfMO)** | Portfólio estratégico | Alta | Comitê de portfólio, diretoria | Gestão de portfólio e alinhamento estratégico |
| **Center of Excellence (CoE)** | Organizacional (conhecimento) | Consultiva | Vice-presidência técnica | Melhores práticas, treinamento, padronização |

---

## 🏢 Enterprise PMO (EPMO)

### Definição e Características

O **Enterprise PMO (EPMO)** é o tipo de PMO com a **maior abrangência possível**. Ele opera no nível corporativo, com autoridade sobre **todos os projetos, programas e portfólios** da organização, independentemente da unidade de negócio ou localização geográfica.

| Característica | Descrição |
| :--- | :--- |
| **Abrangência** | Organizacional total — toda a empresa |
| **Nível hierárquico** | Estratégico — reporta-se à alta liderança (CEO, diretoria executiva) |
| **Autoridade** | Alta — define prioridades, aloca recursos estratégicos |
| **Escopo** | Gestão de portfólio corporativo, governança, alinhamento estratégico |

### Funções Principais

| Função | Descrição |
| :--- | :--- |
| **Gestão de portfólio corporativo** | Seleciona, prioriza e monitora todos os projetos da organização |
| **Alinhamento estratégico** | Garante que projetos estejam alinhados aos objetivos estratégicos |
| **Governança corporativa** | Estabelece políticas, padrões e processos para toda a organização |
| **Alocação de recursos** | Gerencia recursos críticos entre projetos e unidades |
| **Gestão de benefícios** | Acompanha a entrega de valor estratégico |
| **Reportes executivos** | Fornece visibilidade consolidada à alta liderança |

### Quando Utilizar

| Cenário | Justificativa |
| :--- | :--- |
| **Organizações maduras** | Cultura de gestão de projetos consolidada |
| **Empresas com múltiplas unidades** | Necessidade de visibilidade e coordenação central |
| **Estratégia orientada a projetos** | Projetos são o principal mecanismo de execução estratégica |
| **Portfólio complexo** | Muitos projetos interdependentes ou concorrentes por recursos |
| **Transformação organizacional** | Mudanças que afetam múltiplas áreas simultaneamente |

### Exemplo

Uma multinacional do setor de tecnologia estabelece um **EPMO** que:
- Reporta-se diretamente ao CEO
- Gerencia o portfólio global de projetos de transformação digital
- Define quais projetos recebem financiamento e recursos
- Monitora todos os projetos com valor acima de US$ 5 milhões
- Estabelece padrões de governança para todas as unidades

## 🏭 Divisional PMO

### Definição e Características

O **Divisional PMO** opera dentro de uma **unidade de negócio, departamento ou região específica**, sem autoridade sobre outras áreas da organização. Ele atende às necessidades particulares daquela divisão, adaptando práticas ao contexto local.

| Característica | Descrição |
| :--- | :--- |
| **Abrangência** | Unidade de negócio, departamento ou região |
| **Nível hierárquico** | Tático/estratégico — reporta-se à vice-presidência da divisão |
| **Autoridade** | Média — dentro de sua unidade de atuação |
| **Escopo** | Projetos da divisão, alinhamento com estratégia local |

### Funções Principais

| Função | Descrição |
| :--- | :--- |
| **Gestão de projetos da divisão** | Supervisiona projetos dentro da unidade de negócio |
| **Padronização local** | Adapta metodologias às necessidades da divisão |
| **Alocação de recursos locais** | Gerencia orçamento e equipes da unidade |
| **Relacionamento com áreas-fim** | Atende às demandas específicas da divisão |
| **Reportes gerenciais** | Fornece visibilidade à liderança da divisão |

### Quando Utilizar

| Cenário | Justificativa |
| :--- | :--- |
| **Organizações descentralizadas** | Unidades de negócio têm autonomia significativa |
| **Necessidades específicas** | Diferentes unidades exigem metodologias ou processos distintos |
| **Estrutura matricial** | Coordenação entre projetos de uma mesma área funcional |
| **PMO em crescimento** | Primeiro passo antes de um EPMO corporativo |

### Exemplo

Uma empresa de bens de consumo com divisões separadas (alimentos, bebidas, higiene) cria um **Divisional PMO** para a área de **Tecnologia da Informação**, que:
- Reporta-se ao CIO (Chief Information Officer)
- Gerencia apenas projetos de TI
- Utiliza metodologias ágeis específicas para desenvolvimento de software
- Aloca recursos de desenvolvimento entre as demandas das unidades de negócio

## 📋 Project PMO

### Definição e Características

O **Project PMO** é um escritório de projetos **temporário**, criado especificamente para a duração de um **projeto ou programa de grande porte**. Ele é desmobilizado após a conclusão do empreendimento.

| Característica | Descrição |
| :--- | :--- |
| **Abrangência** | Projeto ou programa específico |
| **Nível hierárquico** | Operacional/tático — reporta-se ao patrocinador ou comitê do projeto |
| **Autoridade** | Variável — alta dentro do escopo do projeto |
| **Escopo** | Execução de um empreendimento específico |

### Funções Principais

| Função | Descrição |
| :--- | :--- |
| **Gestão integrada do projeto** | Coordena todas as áreas envolvidas no empreendimento |
| **Planejamento detalhado** | Desenvolve e mantém cronograma, orçamento, recursos |
| **Controle e monitoramento** | Acompanha progresso, riscos, issues |
| **Gestão de mudanças** | Controla alterações de escopo |
| **Comunicação e reportes** | Mantém stakeholders informados |
| **Encerramento** | Documenta lições aprendidas e realiza transição |

### Quando Utilizar

| Cenário | Justificativa |
| :--- | :--- |
| **Projetos de grande porte** | Complexidade exige estrutura dedicada |
| **Projetos críticos** | Alto impacto estratégico ou financeiro |
| **Projetos complexos** | Múltiplas áreas, equipes, fornecedores |
| **Programas de transformação** | Iniciativas com duração e recursos significativos |

### Exemplo

Uma empresa de infraestrutura inicia a **construção de uma usina hidrelétrica** com investimento de R$ 5 bilhões. Um **Project PMO** é criado exclusivamente para este empreendimento, com:
- Equipe dedicada de 20 profissionais
- Reporte direto ao diretor do projeto
- Orçamento próprio e estrutura física no local da obra
- Desmobilização prevista para 6 meses após a conclusão

## 📊 Portfolio Management Office (PfMO)

### Definição e Características

O **Portfolio Management Office (PfMO)** é especializado na **gestão de portfólio** — o conjunto de projetos, programas e outras iniciativas que concorrem por recursos limitados e devem ser selecionadas e priorizadas com base em critérios estratégicos.

| Característica | Descrição |
| :--- | :--- |
| **Abrangência** | Portfólio estratégico (pode ser corporativo ou divisional) |
| **Nível hierárquico** | Estratégico — reporta-se ao comitê de portfólio ou diretoria |
| **Autoridade** | Alta — decide priorização, seleção e cancelamento de iniciativas |
| **Escopo** | Seleção, priorização, balanceamento e monitoramento de portfólio |

### Funções Principais

| Função | Descrição |
| :--- | :--- |
| **Seleção de iniciativas** | Define critérios e seleciona projetos para o portfólio |
| **Priorização** | Ordena iniciativas com base em valor estratégico e recursos disponíveis |
| **Balanceamento** | Equilibra o portfólio entre riscos, horizontes de tempo e tipos de investimento |
| **Alocação de recursos** | Distribui recursos entre projetos concorrentes |
| **Monitoramento de valor** | Acompanha se o portfólio está entregando os benefícios esperados |
| **Governança de portfólio** | Estabelece processos de revisão e decisão |

### Quando Utilizar

| Cenário | Justificativa |
| :--- | :--- |
| **Demanda excede capacidade** | Mais projetos do que recursos disponíveis |
| **Concorrência por recursos** | Projetos competem por pessoas, orçamento ou infraestrutura |
| **Decisões estratégicas** | Necessidade de alinhar investimentos à estratégia |
| **Gestão de valor** | Foco em retorno sobre investimento e benefícios |

### Exemplo

Uma seguradora estabelece um **PfMO** que:
- Recebe todas as propostas de projetos da organização
- Avalia cada iniciativa com base em critérios de valor estratégico
- Define quais projetos entram no portfólio anual
- Balanceia entre projetos de inovação, manutenção e transformação
- Revisa o portfólio trimestralmente, cancelando projetos de baixo desempenho

## 🎓 Center of Excellence (CoE)

### Definição e Características

O **Center of Excellence (CoE)** é um PMO com foco em **conhecimento, padronização e melhores práticas**, sem autoridade direta sobre projetos. Ele atua como um centro de referência, capacitando a organização a executar projetos de forma mais eficaz.

| Característica | Descrição |
| :--- | :--- |
| **Abrangência** | Organizacional ou divisional — foco em conhecimento |
| **Nível hierárquico** | Consultivo — reporta-se a liderança técnica ou de operações |
| **Autoridade** | Baixa (consultiva) — recomenda, não impõe |
| **Escopo** | Metodologias, treinamento, ferramentas, melhores práticas |

### Funções Principais

| Função | Descrição |
| :--- | :--- |
| **Metodologias e padrões** | Define e mantém frameworks de gestão de projetos |
| **Treinamento e capacitação** | Desenvolve competências da comunidade de projetos |
| **Ferramentas** | Seleciona, disponibiliza e mantém softwares de gestão |
| **Lições aprendidas** | Centraliza conhecimento de projetos anteriores |
| **Mentoria e coaching** | Apoia gerentes de projeto em desenvolvimento |
| **Avaliação de maturidade** | Mede e promove evolução da gestão de projetos |

### Quando Utilizar

| Cenário | Justificativa |
| :--- | :--- |
| **Organizações descentralizadas** | Projetos executados por diversas áreas; necessidade de padrões comuns |
| **Cultura de autonomia** | Equipes têm liberdade; PMO como facilitador |
| **Baixa maturidade** | Organização precisa desenvolver competências |
| **Transformação metodológica** | Migração para novas abordagens (ex: ágil) |

### Exemplo

Uma empresa de varejo estabelece um **CoE de Gerenciamento de Projetos** que:
- Define o framework híbrido (água em cascata + Scrum) para toda a organização
- Oferece treinamentos em Microsoft Project, Jira e metodologias ágeis
- Mantém um repositório de templates e lições aprendidas
- Certifica internamente gerentes de projeto
- Atua como consultoria interna para projetos complexos, sem assumir a execução

## 📈 Comparativo entre os Tipos por Abrangência

| Critério | EPMO | Divisional PMO | Project PMO | PfMO | CoE |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Abrangência** | Organizacional | Unidade/Departamento | Projeto específico | Portfólio | Organizacional (conhecimento) |
| **Temporalidade** | Permanente | Permanente | Temporário | Permanente | Permanente |
| **Nível Hierárquico** | Estratégico | Tático/Estratégico | Operacional | Estratégico | Consultivo |
| **Reporte** | CEO/Diretoria | VP da divisão | Patrocinador | Comitê de portfólio | VP técnico |
| **Autoridade** | Alta | Média | Variável (alta no escopo) | Alta | Baixa (consultiva) |
| **Foco Principal** | Alinhamento estratégico | Execução da divisão | Entrega do projeto | Valor do portfólio | Excelência técnica |

## 🏛️ Estruturas Híbridas e Combinações

Na prática, muitas organizações adotam **estruturas híbridas** que combinam diferentes tipos de PMO:

### Modelo Federado

```
                    ┌─────────────────────┐
                    │   EPMO Corporativo  │
                    │  (estratégia,       │
                    │   governança)       │
                    └──────────┬──────────┘
                               │
        ┌──────────────────────┼──────────────────────┐
        │                      │                      │
        v                      v                      v
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│ Divisional    │    │ Divisional    │    │ Divisional    │
│ PMO (TI)      │    │ PMO (Operações)│   │ PMO (Marketing)│
└───────────────┘    └───────────────┘    └───────────────┘
```

**Características:**
- EPMO define padrões, governança e gestão de portfólio corporativo
- PMOs divisionais executam, adaptam padrões e gerenciam projetos locais
- Combina centralização estratégica com autonomia tática

### Modelo com CoE

```
┌─────────────────────────────────────────────────┐
│                  EPMO Corporativo                │
│  (gestão de portfólio, governança, alinhamento) │
└─────────────────────┬───────────────────────────┘
                      │
        ┌─────────────┴─────────────┐
        │                           │
        v                           v
┌───────────────┐           ┌───────────────┐
│ CoE           │           │ Divisional    │
│ (metodologias,│           │ PMOs          │
│  treinamento) │           │ (execução)    │
└───────────────┘           └───────────────┘
```

**Características:**
- CoE concentra conhecimento, metodologias e capacitação
- EPMO foca em estratégia e portfólio
- PMOs divisionais executam projetos

## 🎯 Como Escolher a Abrangência Adequada

A escolha da abrangência do PMO deve considerar múltiplos fatores organizacionais:

### 1. Tamanho e Estrutura da Organização

| Estrutura | Abrangência Recomendada |
| :--- | :--- |
| **Pequena empresa (< 500 funcionários)** | Project PMO ou EPMO simplificado |
| **Média empresa (500-5.000)** | EPMO ou Divisional PMO |
| **Grande empresa (> 5.000)** | EPMO + Divisional PMOs (modelo federado) |
| **Multinacional** | EPMO global + PMOs regionais/divisionais |

### 2. Maturidade em Gerenciamento de Projetos

| Maturidade | Abrangência Recomendada |
| :--- | :--- |
| **Baixa** | CoE (para desenvolver competências) ou Project PMO (projetos críticos) |
| **Média** | Divisional PMO ou EPMO inicial |
| **Alta** | EPMO + PfMO + CoE (modelo completo) |

### 3. Cultura Organizacional

| Cultura | Abrangência Recomendada |
| :--- | :--- |
| **Centralizada, hierárquica** | EPMO, PfMO (controle central) |
| **Descentralizada, autônoma** | CoE, Divisional PMOs (autonomia) |
| **Orientada a projetos** | EPMO com alta autoridade |

### 4. Complexidade do Portfólio

| Complexidade | Abrangência Recomendada |
| :--- | :--- |
| **Baixa (poucos projetos)** | Project PMO ou CoE |
| **Média (dezenas de projetos)** | Divisional PMO |
| **Alta (centenas de projetos)** | EPMO + PfMO |

## 📝 Considerações Finais

A classificação do PMO por abrangência é fundamental para definir o escopo, autoridade e estrutura do escritório de projetos em uma organização.

**Principais aprendizados:**

1. **Abrangência define autoridade** — quanto maior a abrangência, maior a autoridade e a visibilidade estratégica do PMO.

2. **Não há modelo único** — a estrutura ideal depende do tamanho, maturidade, cultura e estratégia da organização.

3. **Modelos híbridos são comuns** — especialmente em grandes organizações, combinam EPMO centralizado com PMOs divisionais.

4. **A evolução é possível** — organizações podem iniciar com um CoE ou Project PMO e evoluir para estruturas mais abrangentes.

5. **Alinhamento estratégico é essencial** — independentemente da abrangência, o PMO deve contribuir para a entrega de valor e o alcance dos objetivos organizacionais.

A escolha da abrangência adequada é um dos fatores críticos para o sucesso de um PMO. Um PMO com abrangência muito ampla pode ser percebido como **burocrático e distante** das necessidades locais; um PMO com abrangência muito restrita pode ser **irrelevante** para a estratégia corporativa. O equilíbrio certo — frequentemente alcançado por meio de estruturas federadas — é a chave para que o PMO agregue valor em todos os níveis da organização.
